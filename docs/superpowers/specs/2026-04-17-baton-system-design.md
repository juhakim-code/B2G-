## 1. 프로젝트 개요

**바톤**은 슬랙 기반의 AI 업무 인수인계 챗봇이다.  
슬랙 채널 기록을 자동으로 분석해 인계 내용을 추출하고, 인계자의 검토를 거쳐 인수자에게 체계적으로 온보딩한다.

### 대상 사용자
- 신입직원 (처음 업무를 배우는 단계)
- 기존 재직자 (담당 업무 변경 또는 인수인계 받는 상황)

### 플랫폼
- 슬랙(Slack) — 기존 업무 환경과 연계

---

## 2. 학습 시스템

### 학습 방식

| 방식 | 내용 | 주기 |
|------|------|------|
| 전체 기록 학습 | 슬랙 업무 채널 전체 기록을 Claude가 읽고 핵심 지식 자동 추출 | 매주 자동 실행 |
| 실시간 학습 | 새 채널 메시지를 실시간 모니터링하여 업무 관련 내용을 지식창고에 즉시 추가 | 상시 |

### 학습 범위
- **포함:** 업무 채널 전체
- **제외:** DM(1:1 대화)

### 지식창고 구조 (해커톤 버전)

```
바톤 지식창고/
  ├── 업무_프로세스.md     ← 비용신청, 휴가 등 절차
  ├── 자주_묻는_질문.md    ← 슬랙에서 반복된 질문들
  ├── 담당자_스타일.md     ← 전임자의 말투·판단 기준
  └── 업데이트_로그.md     ← 언제 무엇이 추가됐는지
```

### 자동 추출 항목 예시
- 반복 업무 프로세스 (비용신청, 출결 처리 등)
- 자주 묻는 질문 (채널에서 반복 등장한 질문)
- 협업 관계 (자주 소통한 담당자·부서)
- 주의사항·예외 (놓치기 쉬운 특이 케이스)

## 3. 사용자 경험 흐름 (3단계)

### 1단계: 세팅
1. 슬랙 연결 및 봇 설치
2. 읽을 채널 선택
3. 첫 전체 학습 실행
4. 바톤 봇 활성화

### 2단계: 온보딩 (핵심)

```
① 바톤이 슬랙 자동 분석
   └── 채널 기록에서 인계 필요 항목 후보 추출

② 인계자가 검토·선택
   └── 후보 목록 중 실제 인계할 항목 승인/제외

③ 인계시트 자동 생성
   └── 바톤이 선택된 내용을 정리된 문서로 작성

④ 인수자에게 능동적 온보딩
   └── 바톤이 슬랙 DM으로 단계별 전달

⑤ 추가 Q&A
   └── 인수자가 궁금한 점을 DM으로 질문 가능
```

### 3단계: 실전 시뮬레이션
- AI 페르소나 봇 20명(강성 포함)이 DM·스레드로 실제처럼 질문
- 담당자가 직접 대응하며 실전 감각 체득
- 대응 결과 기록 및 분석

## 4. 기술 구성

### 해커톤 버전 (현재)
| 컴포넌트 | 기술 |
|----------|------|
| AI 엔진 | Claude API |
| 슬랙 연동 | Slack API (봇 토큰) |
| 지식창고 | 마크다운 파일 |
| 주기 실행 | 스케줄러 (cron) |

### 실서비스 버전 (이후)
| 컴포넌트 | 기술 |
|----------|------|
| AI 엔진 | Claude API |
| 슬랙 연동 | Slack API (웹훅) |
| 지식창고 | 벡터 DB (Supabase) |
| 실시간 처리 | 웹훅 이벤트 기반 |
